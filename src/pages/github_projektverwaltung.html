Projektverwaltung – GitHub

Beim Anlegen eines neuen Projektes muss entweder eines von Grund auf neu einrichten oder aber auch ein bereits bestehendes von GitHub herunterladen um daran weiterzuentwickeln.

Dazu braucht man zuerst mal einen Account bei GitHub wo man eben das ganze Projekt verwalten kann und auch immer wieder auf ältere Versionen zurückgreifen kann, wenn man schön sauber damit arbeitet.

Neues, (bereits bestehendes) Projekt verwalten

  1. Zuerst geht man auf GitHub (The world's leading software development platform ·...) und wählt dort das entsprechende Projekt aus, an welchem man weiterentwickeln möchte.
  2. Dann klont man das Projekt, indem man, wenn man auf diesen Button geklickt hat, man den Link kopiert und diesen dann bei der Desktopanwendung von GitHub einfügt um das Projekt lokal zu speichern. Um das Projekt dort einzufügen geht man über den Menüpunkt File > Clone Repository, bestimmt den Ablageort und klont das Ganze.
  3. Danach muss man noch die Javascript Programme installieren, welche man zum Entwickeln benötigt. Dies kann man über die Command-Line tun:
    1. In den Projektordner navigieren mit Hilfe folgender Befehle:
      ls
      cd (Ordnername)
      pwd
    2. die Installationspakete herunterladen:
      npm install
      bower install
  4. Um das Ganze noch etwas sicherer zu machen, bevor man anfängt zu programmieren, kann man sich einen neuen Ast (Branch) erstellen, damit man auf dem Master nichts verpfuscht. Und kann diesen dann später, wenn man sicher ist, mit dem Masterbranch vereinen.
  5. Danach die Browseransicht des Projektes starten:
    foundation watch
  6. Den Texteditor zum Beispiel Atom starten und das Projekt darin öffnen und anfangen zu programmieren.
  7. Nach den Bearbeitungen foundation watch beenden:
    exit
  8. Wenn man die Änderungen vorgenommen hat und denkt, dass das Resultat verhält, dies auf GitHub über die Desktopanwendung commiten und damit bestätigen, dass man die Änderungen in das Lokaleprojekt integrieren möchte.
  9. Beim Commiten ist es sehr hilfreich, wenn man einen Kommentar zu den Änderungen dieser Version macht, da dies sehr hilfreich bei allfälligen Problemen oder ähnlichem sein kann und man sich so viel Zeit ersparen kann, bei beispielsweise einer Fehlerbehebung.
  10. Danach kann man die neue lokale Version pushen und somit an den Origin weitergeben, wo dann entschieden werden kann, ob man diese Version in die aktive integrieren möchte.

Weiterarbeit an einem Projekt

  1. Bei der Weiterarbeit an einem Projekt, sollte man die aktuellste Version davon fetchen.
  2. Und muss dann wieder die Installationspakete herunterladen, da diese nicht im Abbild niintegriert werden; was Sinn macht, da diese immer auf dem neusten Stand sein sollten.
  3. Danach startet man auch wieder die Browseransicht des Projektes.
  4. Wenn es im Projekt durch Änderungen zu Überlappungen kommt, muss man für diese eine Lösung finden und schauen welche Version man jetzt verwenden möchte.
  5. Das weitere Vorgehen ist wie oben schon beschrieben Punkt 8 - 11.

Aufbereiten für den Webbrowser – komprimieren

  1. Auf der Command-Line in den Projektordner gehen und zum komprimieren des Projektes folgenden Befehl eingeben:
    foundation build
  2. Dann FileZilla starten und sich mit dem FTP-Server verbinden.
  3. Dort den Inhalt aus dem 'dist'-Ordner kopieren und auf den Server laden.
